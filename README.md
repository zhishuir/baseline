# 基于自编码器的网络安全攻击基线方法与实现

面向网络入侵检测的深度学习基线检测系统：用 Transformer 增强的自编码器学习正常流量基线，用 k-Sigma 滑动窗口动态阈值判决基线偏离，在 CIC-IDS2018 数据集上实现 15 类攻击的细粒度识别与未知攻击检测。

> 本仓库只包含论文自身的方法（不含企业需求驱动的多源日志融合演示，那部分是独立系统，未包含在本仓库中）。

## 特性

- **多任务 Transformer 增强自编码器**——编码瓶颈嵌入 3 层 8 头 Transformer，建模流量特征间的全局依赖；重构、15 类分类、二分类三任务联合优化，兼具生成式与判别式能力
- **k-Sigma 滑动窗口动态阈值**——Welford 增量算法 O(1) 维护均值方差，判决阈值随基线分布实时漂移调整；攻击样本隔离机制从根源阻断"漏报→漂移→更多漏报"的正反馈
- **类别不均衡三层协同处理**——数据层加权采样、特征层 Mixup 增强、梯度层 Focal Loss
- **未知攻击检测**——基于重构误差与分类置信度的联合信号，识别训练分布外的攻击类型
- **可视化演示系统**——内置浏览器界面，实时推理单条样本、查看动态阈值与分类概率

## 核心结果

在 CIC-IDS2018 数据集（94,497 条记录，15 类，测试集 9,450 条）上：

| 指标 | 数值 |
|---|---|
| 整体准确率 | 84.0% |
| 加权 F1 | 0.838 |
| 二分类 AUC / AP | 0.9395 / 0.9415 |
| 攻击召回率（TPR） | 87.6% |
| 误报率（k-Sigma 动态阈值 vs 固定阈值） | 8.7%（较固定阈值 12.4% 相对下降 30%） |
| 未知攻击平均检出率（4 类留出攻击） | 82.7%（较单任务基线提升 24.8 个百分点） |

详细的逐类别指标、消融实验、协同增益分析见 `工程实现补充材料.docx`。

## 安装

```bash
pip install torch numpy pandas scikit-learn
```

GPU 可选（CUDA 环境自动启用），CPU 环境也能运行，仅推理速度较慢。

## 使用

### 网页演示

```bash
python web_demo.py
```

浏览器打开 `http://localhost:8090`，三个标签页：

- **系统概览**——模型信息、两项核心创新说明、异常检测公式
- **单条检测**——拖动滑块选择测试样本，实时推理，查看异常得分 / 动态阈值 / 15 类概率分布
- **论文指标**——15 类分类结果明细表（精确率 / 召回率 / F1 / 样本数）

### 训练

```bash
python train.py
```

需要完整的原始 CIC-IDS2018 数据集（约 2.7GB，未包含在本仓库，需从 [CIC 官网](https://www.unb.ca/cic/datasets/ids-2018.html) 自行下载），训练配置见 `train.py` 顶部 `CONFIG` 字典（3 层 8 头 Transformer、批大小 256、AdamW、800 epoch）。

## 方法概览

```
30维流量特征 → StandardScaler标准化 → 残差编码器(30→512) → 3层8头Transformer
→ 512维潜在基线表征 → 残差解码器(512→30) + 15类分类头 + 二分类头
→ 复合异常得分 s = 0.7×重构误差 + 0.3×攻击概率
→ k-Sigma动态阈值判决 (θ = μ + 1.5σ，滑动窗口 + 攻击样本隔离)
→ 正常 / 已知攻击 / 疑似未知
```
